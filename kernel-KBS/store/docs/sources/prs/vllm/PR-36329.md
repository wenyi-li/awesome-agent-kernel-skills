## Summary

Fixes #35924 — Qwen3.5 `GatedDeltaNet` `in_proj_ba` fails Marlin `MIN_THREAD_N=64` at TP>=2.

**Root cause:** `in_proj_ba` uses `MergedColumnParallelLinear` with `output_sizes=[num_v_heads] * 2`. When TP-sharded, the per-rank output falls below Marlin's minimum (e.g., 397B: `64 / TP=4 = 16 < 64`).

**Fix:** Replace with two separate `ReplicatedLinear` layers (`in_proj_b`, `in_proj_a`) that preserve `quant_config`. Each rank computes the full projection and slices to its local TP partition in `forward()`.
