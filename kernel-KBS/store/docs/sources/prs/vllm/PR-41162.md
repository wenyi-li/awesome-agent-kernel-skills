## Summary

## Context
Investigating DSV4 "invalid memory access" crash that happens when MTP > 2. I believe that crash is caused by not rebuilding the attention metadata in between draft decode steps. DSV4's attention metadata builders have position-dependent state that must be updated whenever the position is advanced. For example: https://github.com/vllm-project/vllm/blob/e9f8f31e9a4c31d6842ca1adffe2619ed204fafb/vllm/v1/attention/backends/mla/sparse_swa.py#L293-L307.

During draft decoding, we only update the sequence lengths, positions, and slot mappings, but not the metadata properties dependent on those: https://github.com/vllm-project/vllm/blob/e9f8f31e9a4c31d6842ca1adffe2619ed204fafb/vllm/v1/worker/gpu/spec_decode/eagle/speculator.py#L316-L327.

Any attention backend with builders that have position-dependent state can potentially be affected by this bug.

## This PR
Fixes the above issue by rebuilding attention metadata in between draft generation steps during the multi-step decode loop.
