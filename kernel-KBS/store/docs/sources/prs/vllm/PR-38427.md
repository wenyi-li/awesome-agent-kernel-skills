## Summary

vLLM PR #38427 changed the batch-invariant Triton matmul dispatch from an exact SM80 match to an SM80-family match. This enabled the Triton replacement on `sm80`, `sm86`, and `sm87`, fixing determinism failures on RTX 3090-class hardware where the path had fallen back to cuBLAS.

## Kernel-KBS Lessons

- Do not use an exact `sm80` predicate when the kernel is intended for all Ampere-family GPUs.
- Test SM86/SM87 paths separately from A100; exact-match dispatch bugs can silently route production traffic to a different backend.
- For determinism-sensitive matmul, verify batch-invariant behavior on the actual Ampere SKU instead of relying only on A100 coverage.
