## Summary

Adds initial CUTLASS kernel support for FP8 GEMM operations on NVIDIA Blackwell (SM100) GPUs in vLLM. Introduces the `sm100_fp8_config_default` configuration with static tile and cluster shapes as the foundational Blackwell FP8 GEMM implementation.

## Problem

vLLM lacked native FP8 GEMM kernel support for the Blackwell architecture. Without SM100-specific kernels, FP8 inference on Blackwell GPUs could not fully utilize the new tensor core generation (tcgen05) and required fallback to less optimized code paths.

## Solution / Techniques

- Implements `sm100_fp8_config_default` CUTLASS configuration for SM100 FP8 GEMM
- Static tile and cluster shapes for initial Blackwell support
- Added default constructors for C2X and C3X kernel variants to prevent build failures
- CUDA version guards (CUDA >= 12.8) to prevent linker errors on older toolchains
- Modified epilogue definitions for Blackwell-specific architectural constraints
- Maintains 128-bit data alignment for optimal TMA performance
- Separate compilation unit for SM100 FP8 operations

## Performance

Initial Blackwell FP8 GEMM implementation providing a foundation for further tuning. Future iterations planned to extend optimization coverage for additional tile configurations and dual-SM GEMM operations. Benchmark results on tcgen05 show competitive FP8 throughput leveraging SM100 tensor cores.
