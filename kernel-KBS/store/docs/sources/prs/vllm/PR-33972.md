## Summary

When `dtype=float16`, due to the smaller dynamic range of float16, data overflow can easily occur during GEMM, leading to inference errors.
Therefore, we dynamically scale the input before GEMM to prevent data overflow.

Fix: https://github.com/vllm-project/vllm/issues/33560 https://github.com/vllm-

## Problem

[Bugfix]fix output Nan/Inf in marlin if dtype=float16
