## Problem

`CuMemAllocator` (sleep-mode allocator, `vllm/device_allocator/cumem.py`) uses a CUDA *pluggable* allocator. When a pool-backed tensor's refcount hits zero, `_python_free_callback` returns the allocation handle, and the C extension (`csrc/cumem_allocator.cpp:unmap_and_release`) immediately calls `cuMemUnmap` + `cuMemRelease` *synchronously from the host*. Which can trigger IMA in the case where the memory is released before all kernels on the cuda stream have completed. 

This has showed up when running deepseek v4 with sleep mode turned on.
