## Summary

## Purpose

Eagle3 checkpoints from NVIDIA (GPT-OSS series) store `norm_before_fc` inside the nested `eagle_config` dict rather than as a top-level config field. We currently only check the top-level, so the RMSNorm before FC gets silently skipped for these checkpoints, causing degraded acceptance rate.

Fix: read `norm_before_fc` from `eagle_config` first, fall back to top-level attr. Same pattern already used for `use_aux_hidden_state` in the same block.
