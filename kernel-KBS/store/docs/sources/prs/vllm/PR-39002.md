## Summary

## Purpose

Fix FlashInfer attention crashing when `kv_cache_dtype_skip_layers` is used (introduced in #33695).

The metadata builder read the global `cache_config.cache_dtype` (e.g., `"fp8"`) instead of the per-group `kv_cache_spec.kv_quant_mode`, so skipped layers were still treated as fp8.
