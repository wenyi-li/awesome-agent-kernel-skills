## Summary

## Purpose
Attention backends all support bfloat16, but only report support for `auto`. In cases where the automatically resolved dtype is not bfloat16, this can lead to failures.

Related issue: #32732 - setting `--kv-cache-dtype=bfloat16` should work as a workaround, but does not, because TRITON_M

## Problem

[Bugfix][Attention] Explicitly report support for kv_cache_dtype bfloat16
