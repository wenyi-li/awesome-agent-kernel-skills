## Summary

DeepSeek V4 Pro Base FP8 routed MoE needs the model-provided SwiGLU clamp limit to be applied before the second expert GEMM. Without this clamp, DSV4 Pro Base produces garbled outputs and very low eval quality.

This PR patches FP8 MoE backends ('auto' and 'triton') as the Base model's moe experts are FP8.

After this PR, we saw mmlu pro bench has macro 0.7409 / micro 0.7298, which matches model card.

Related PR for instr model's FP4 path (#42287, #42541)

## Problem
