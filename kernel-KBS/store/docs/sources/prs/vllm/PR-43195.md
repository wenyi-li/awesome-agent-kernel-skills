## Summary
- Convert KDA chunk cumulative gates to base-2 log space and use `exp2` in the KDA chunk kernels that consume those cumulative gates.
- Add an opt-in `use_exp2` path to the shared `chunk_delta_h` kernel, leaving the default `exp` behavior unchanged for non-KDA callers.
- Add a reproducible `benchmarks/kernels/benchmark_kda.py` script for KDA chunk prefill latency and nsys profiling.

## Why this PR
KDA's mathematical recurrence is still natural-base decay, i.e. `state *= exp(g)`. This is why recurrent/decode paths keep using `exp`: those paths consume per-step gates directly and do not convert the gate representation.

The chunk/prefill path is different. Official FLA moved the optimized KDA chunk kernels to the standard `RCP_LN2 + exp2` implementation in https://github.com/fla-org/flash-linear-attention/pull/679: the chunk-local cumulative gate is scaled by `1 / ln(2)`, and the chunk kernels that consume that cumulative gate use `exp2`. This preserves the original math because `exp(x) == exp2(x / ln(2))`, while using the faster base-2 exponential path inside Triton kernels.
