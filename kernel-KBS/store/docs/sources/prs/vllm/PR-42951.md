## Summary
Add pure PyTorch fallback for MoE `sqrtsoftplus` scoring on XPU (no Triton dependency), and register XPU in MXFP4 backend priority list.

## Changes
- **fused_topk_bias_router.py**: add `_topk_softplus_sqrt_torch()` pure PyTorch fallback for `sqrtsoftplus` scoring on non-CUDA platforms
- **mxfp4.py**: add XPU to priority backend list with no-op weight conversion

## Testing
Tested with DeepSeek-V4 MoE routing on Intel XPU.

## Notes
- No functional change for CUDA path (Triton kernel still used)
- Part of DeepSeek-V4 XPU support series (2/4)
