## Summary

# Context
Per-position acceptance rates were recently added in [this PR](https://github.com/vllm-project/vllm/pull/40662), adding support for both MRV1 and MRV2. However, MRV2 currently uses [a separate path](https://github.com/vllm-project/vllm/blob/a0dc7a0f368a3f212d37e39ba4dd7b682e378fe2/vllm/v1/worker/gpu/spec_decode/rejection_sampler.py#L140-L152) for synthetic rejection sampling, which is not ideal because synthetic acceptance rates should closely approximate actual rejection sampling.

# This PR
I apply the synthetic mode in the probabilistic rejection sampling kernels. This involved pass the synthetic conditional rates to `_probabilistic_rejection_kernel` (now renamed to `_rejection_kernel`). If non-None, the draft token is accepted with the conditional probability of that given step. Note that the kernel still loads target/draft logits, sum-exponentials, etc, even if unused to closely match the performance of actual rejection sampling.

Other notable changes:
- Deleted `synthetic_rejection_sampler_utils.py`, not used anymore.
- Renamed `probabilistic_rejection_sampler_utils.py` to `rejection_sampler_utils.py`, as well as removed `probabilistic` prefix from all meth
