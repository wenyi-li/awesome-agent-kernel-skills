## Summary

Alternative to #39727. 
Interestingly, current nvfp4 cutlass linear is already batch invariant, although possibly by accident. 
This PR adds an explicit flag that makes it use a dedicated cutlass config and a static assert to prevent someone from accidentally breaking it by changing the scheduler.
In my synthetic ad-hoc benchmarks it matches normal cutlass mode in most scenarios and beats the triton kernel everywhere:

```
--- infer_1x8192x8192 ---
shape M=1 N=8192 K=8192 dtype=bf16  warmup=5 iters=20
Triton (batch-invariant): median 0.0553 ms  min 0.0534 ms  -> 2.43 TFLOP/s (median)
CUTLASS:                  median 0.0384 ms  min 0.0368 ms  -> 3.49 TFLOP/s (median)
CUTLASS (batch-invariant): median 0.0453 ms  min 0.0432 ms  -> 2.96 TFLOP/s (median)
NVFP4 emulation (PyTorch, bf16 matmul): median 1.9479 ms  min 1.9363 ms  -> 0.07 TFLOP/s (median)
