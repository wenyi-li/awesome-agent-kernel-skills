## Summary

Fixes #41651. And also fixes the illegal memory access error when serving that model with certain configs.

The FlashInfer and Triton attention metadata builders source `num_qo_heads` / `num_heads_q` from `model_config.get_num_attention_heads(parallel_config)`, which returns the model-wide value. Models that override the head count on a subset of layers (Laguna XS.2 sets `num_attention_heads_per_layer = [48, 64, 64, 64, ...]`) end up with one of their kv-cache groups under-sized at plan time, while the kernel still reads the actual per-layer head count from `q.shape[1]` at runtime. The two values disagree and the kernel produces wrong output.

This change reads `num_heads` directly off each served `Attention` layer instance (which the model has already populated with the per-layer, per-TP-rank value) and asserts uniformity within the attention group. The shared helper lives in `vllm/v1/attention/backends/utils.py` next to `get_per_layer_parameters`; both backends call it through one line.
