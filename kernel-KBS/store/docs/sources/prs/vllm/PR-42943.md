## Summary

## Purpose
Following #40119, extend the RVV attention kernel from VLEN=128-only to support both VLEN=128 and VLEN=256(via #39478).

> -march=rv64gcv_zvfh_zvl256b -mrvv-vector-bits=zvl -DRISCV_FP16_SUPPORT (VLEN=256) — the RVV kernel is correctly omitted and dispatch falls back to VEC.

#40119 currently falls back VLEN=256bit to VEC.

## Test Plan
Compile with -DVLLM_RVV_VLEN=256 on Spacemit X100 (VLEN=256) 

## Test Result
Benchmarked on Spacemit X100 (VLEN=256), BFloat16, GQA 32Q/8KV, head_size=128:

| Scenario | VEC (baseline) | RVV (optimized) | Speedup |
| :--- | :--- | :--- | :--- |
| Decode (bs=32, q=1, kv=2048) | 23.58 ms | 20.54 ms | **1.15×** |
| Prefill (bs=4, q=512, kv=512) | 231.41 ms | 76.89 ms | **3.01×** |
| Mixed (bs=16, q=mix, kv=mix) | 1498.77 ms | 418.41 ms | **3.58×** |
