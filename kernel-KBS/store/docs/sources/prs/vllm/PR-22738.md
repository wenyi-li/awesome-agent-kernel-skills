## Summary

Fixes a bug where the CUTLASS MLA attention backend was not being automatically selected on SM100 (Blackwell) devices despite being the intended default. The issue stemmed from cached reads in the attention backend selector that prevented detection of updated environment variables, causing Triton MLA to be chosen instead on tcgen05 GPUs.

## Problem

On tcgen05 (Blackwell SM100) hardware, vLLM incorrectly selected the Triton MLA backend instead of the faster CUTLASS MLA backend. The attention backend selector in `vllm/attention/selector.py:get_attn_backend` cached results that prevented proper detection of the SM100 device capability, even when the CUTLASS MLA backend was explicitly forced via environment variables.

## Solution / Techniques

- Modifies the attention backend selection logic to rely on device capability and block size checks rather than cached environment variable reads
- Ensures SM100 devices automatically get the CUTLASS MLA backend without manual configuration
- More robust backend detection that correctly identifies Blackwell hardware
