## Summary

## Purpose
Move the V4 sparse-MLA forward path out of vllm.v1.attention.backends and into vllm/models/deepseek_v4/attention/. DeepseekV4MLAAttention now picks its CUDA or ROCm impl once at __init__ via a single platform check inside _select_v4_sparse_impl(); forward() and get_attn_backend() are platform-agnostic.

- New abstract DeepseekV4SparseMLAAttentionImpl with two sibling subclasses: DeepseekV4FlashMLASparseImpl (CUDA) and DeepseekV4ROCMAiterMLASparseImpl (ROCm). Both expose forward_mqa as a layer-driven classmethod. PREFILL_CHUNK_SIZE is a ClassVar on the abstract base.
- DeepseekV4FlashMLASparseBackend moves from flashmla_sparse.py into the V4 impl package. The V3.2 base now declares head_size [576] only; V4 overrides with [512]. Both V4 backends share the 584B fp8_ds_mla cache shape.
- Warmup no longer needs a wrapper-level early return + dummy workspace pre-reservation. forward_mqa detects None attn_metadata, reserves the bf16 gather workspace at max size directly, and returns before the dequantize / topk / sparse_fwd chunk-loop kernels would launch.
- Folder: vllm/models/deepseek_v4/attention/{__init__.py, deepseek_v4_attention.py, impl/{__init__.py, flashmla.py, r
