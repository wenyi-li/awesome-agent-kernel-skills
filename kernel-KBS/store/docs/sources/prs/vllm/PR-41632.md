## Summary

## Purpose

`benchmarks/benchmark_serving_structured_output.py` currently measures structured-output serving workloads, but it does not provide a way to add a shared prompt prefix across requests.
This makes it harder to benchmark structured-output workloads that also exercise prefix-cache behavior or long shared-prefix behavior.

The same option already exists in `vllm bench serve`. Reusing the option name from here.

A similar option exists in `benchmarks/benchmark_prefix_caching.py` through `--prefix-len`, where the benchmark generates one common random token prefix and prepends it to all sampled prompts.

## Test Plan

* Verify --help shows the new option.
* Run the benchmark with the default value and confirm behavior is unchanged.
* Run a small benchmark with --random-prefix-len and confirm prompt lengths increase by the requested prefix length.
