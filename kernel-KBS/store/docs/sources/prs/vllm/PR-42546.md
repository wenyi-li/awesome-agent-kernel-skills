## Summary

## Purpose

  Add Qwen3.5/3.6 VLM checkpoint prefix mapping for ModelOpt mixed-precision checkpoints.

  Qwen VLM checkpoints can export language model tensors and quantization config entries with wrapper prefixes such
  as `model.language_model.*` or `language_model.model.*`, while vLLM constructs the text model modules under
  normalized internal prefixes such as `model.*` / `lm_head.*`. 

  This PR adds the required mapping so:

  - Qwen3.5/3.6 VLM checkpoint weights load into the expected vLLM module names.
  - ModelOpt `quantized_layers` entries resolve correctly when wrapper prefix ordering differs.
  - Nested VLM `lm_head` prefixes resolve to top-level `lm_head` quantization metadata when lm_head quantization support is present. (related PR: https://github.com/vllm-project/vllm/pull/42124)

  ## Test Plan

  - Run smoke generation with a Qwen3.6 VLM ModelOpt mixed-precision checkpoint.
  - Verify ModelOpt `quantized_layers` resolves the expected quantized modules when checkpoint prefixes use
  `model.language_model.*` / `language_model.model.*`.
  - Verify text-only loading works with Qwen VLM architecture override and disabled multimodal inputs.
