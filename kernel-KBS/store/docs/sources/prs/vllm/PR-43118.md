## Summary

`AyaVisionForConditionalGeneration` and `VoxtralForConditionalGeneration` (and, by inheritance, `VoxtralRealtimeGeneration`) declare the `SupportsPP` mixin but never wire `self.make_empty_intermediate_tensors` from `self.language_model`. The attribute then resolves to the stub method inherited from the `SupportsPP` Protocol — its body is `...`, so it returns `None`.

This is subtle: `@runtime_checkable` only verifies the attribute exists, so `isinstance(model, SupportsPP)` passes through the inherited stub. `supports_pp()` returns `True` and the engine enables PP. On non-first PP ranks, [`gpu/model_runner.py:322`](https://github.com/vllm-project/vllm/blob/main/vllm/v1/worker/gpu/model_runner.py#L322) does

```python
self.intermediate_tensors = self.model.make_empty_intermediate_tensors(...)
```

gets `None`, and crashes downstream. Same exact failure mode that #42819 fixed for Cohere2 Vision.

The fix is the one-line forward that the rest of the VL wrappers already use (see `eagle2_5_vl.py`, `internvl.py`, `lfm2_vl.py`, `skyworkr1v.py`, …):

```python
self.make_empty_intermediate_tensors = (
    self.language_model.make_empty_intermediate_tensors
)
```
