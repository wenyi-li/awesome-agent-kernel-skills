## Summary

## Purpose

This change adds truncation_side support to vLLM’s OpenAI-compatible /v1/completions and /v1/chat/completions endpoints.
  When truncate_prompt_tokens is set, callers can now explicitly choose whether vLLM keeps the beginning or the end of the
  prompt.

  "right" keeps the first N prompt tokens, while "left" keeps the last N prompt tokens.
