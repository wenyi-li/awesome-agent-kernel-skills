## Summary

# Context
Currently in MRV2, we are rebuilding the attention metadata before replaying the FULL cudagraph draft prefill instead of reusing the target model's attention metadata. This guards for situations where the draft model's attention metadata builder has state that was captured during FULL cudagraph. An example of this can be found [here](https://github.com/vllm-project/vllm/blob/5544f8c18b9e9ae20e41e019d23d58260940f225/vllm/v1/attention/backends/flash_attn.py#L355) in the FlashAttention backend. That scheduler metadata buffer is passed into the built attention metadata, and is baked into the cudagraph.

When that happens, we must rebuild the attention metadata using the same builder as during capture, otherwise the draft prefill will run with stale data, which  in some cases can cause crashes.

# This PR
The key changes in this PR are passing the same attention metadata used by the target model during cudagraph capture (in `ModelCudaGraphManager.capture`) to the speculator during capture (`PrefillEagleCudaGraphManager`). This is crucial, because it avoids rebuilding the attention metadata again with the speculator's builders.
