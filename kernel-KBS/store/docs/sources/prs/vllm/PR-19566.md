## Summary

Implements performance tuning for the SM100 FP8 CUTLASS GEMM kernel in vLLM by introducing M-dimension-dependent configuration dispatch. Different CUTLASS tile and cluster configurations are selected based on the batch dimension to maximize throughput across varying workload sizes on tcgen05 hardware.

## Problem

The initial SM100 FP8 CUTLASS kernel (from PR #13798) used a single static configuration for all matrix sizes. This left performance on the table for medium and large batch sizes where different tile and cluster shapes could better utilize Blackwell's SM100 tensor cores and memory hierarchy.

## Solution / Techniques

- Introduces M-dimension-dependent dispatch in `scaled_mm_sm100_fp8_dispatch.cuh`
- Three configuration tiers based on matrix M dimension:
  - M <= 128: existing configuration maintained
  - 128 < M <= 256: new `sm100_fp8_config_M256` with adjusted tile/cluster shapes
  - M > 256: enhanced `sm100_fp8_config_default` with refined parameters
- Each configuration optimizes tile shapes and cluster dimensions for its target range

## Key Code

```cpp
// M-dimension dependent dispatch for SM100 FP8 GEMM
if (m <= 128) {
    // Use existing sm100_fp8_config for small batches
} else if (m <= 256) {
    // sm100_fp8_config_M256: optimized tile/cluster for medium batches
} else {
    // Enhanced sm100_fp8_config_default for large batches
}
```

## Performance

Benchmarked on tcgen05 with Llama-3.1-8B-Instruct shapes (N values: 4096, 6144, 14336, 28672):

| M Dimension | Speedup Range |
|-------------|---------------|
| M=128 | up to 1.32x |
| M=256 | up to 1.41x |
| M>=512 | up to 1.45x |

Significant improvements across FP8-channel and FP8-tensor quantization modes, with the largest gains at medium-to-large batch sizes.
