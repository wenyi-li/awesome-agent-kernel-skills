## Summary

## Purpose

Some accelerators (e.g. custom NPUs) require vLLM to leave DeviceConfig.device unset so that the host CPU handles input pre-processing, the same way TPU already works.

- `Platform.uses_cpu_device()` defaults to `self.is_tpu()` so existing behaviour is unchanged; platforms that need CPU-side input processing can override it to return `True`
- `DeviceConfig.__post_init__` checks the hook in addition to the existing hardcoded `"tpu"` string, making it open for extension

## Test Plan

Test verifies that monkeypatching the hook produces the expected `None` device value
`pytest -q tests/test_config.py -k 'device_config_uses_platform_cpu_device_hook'`
