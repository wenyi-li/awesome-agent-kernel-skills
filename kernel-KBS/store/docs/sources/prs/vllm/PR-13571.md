## Summary

Implements FP4 (4-bit floating point) GEMM kernels using CUTLASS 3.8 for NVIDIA Blackwell (SM100+) GPUs in vLLM. This is the inference kernel component of NVFP4 support, separated from the quantization logic for cleaner code organization. Enables scaled matrix multiplication with FP4 quantized weights.

## Problem

Blackwell GPUs natively support NVFP4 (4-bit floating point) computation, but vLLM lacked GEMM kernels to exploit this capability. FP4 quantization can halve memory requirements compared to FP8, enabling larger models or higher throughput. The CUTLASS 3.8 library introduced the necessary primitives for SM100 FP4 GEMM.

## Solution / Techniques

- CUTLASS 3.8-based kernel wrappers for FP4 scaled matrix multiplication
- Targets compute capability 100+ (Blackwell SM100)
- Requires CUDA 12.8+
- Dual-scale input handling with A_sf and B_sf scaling factor parameters
- Explicit kernel instantiations rather than full templating (CUTLASS 3.8 compatibility)
- Architecture-specific configuration structs with device dispatch
- Fallback error handling for unsupported CUTLASS versions

## Key Code

```cpp
// NVFP4 CUTLASS GEMM for SM100 (Blackwell)
// Supports FP4 quantized weights with dual scaling factors
void tcgen05_scaled_mm_sm100_fp4(
    torch::Tensor& out,        // float/half/bfloat16 output
    torch::Tensor const& a,    // FP4 input activation
    torch::Tensor const& b,    // FP4 weight
    torch::Tensor const& a_sf, // activation scaling factor
    torch::Tensor const& b_sf  // weight scaling factor
);
```

## Performance

Enables 4-bit inference on Blackwell GPUs with approximately 2x memory reduction compared to FP8 quantization. The CUTLASS 3.8 FP4 kernels leverage SM100's native NVFP4 tensor core support for high-throughput matrix multiplication at 4-bit precision.
