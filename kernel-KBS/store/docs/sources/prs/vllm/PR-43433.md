## Summary

## Summary

- keep scheduler liveness true while a finished request is waiting on delayed KV connector cleanup
- yield briefly on no-forward engine steps when scheduler work remains, so background connector progress can complete
- add scheduler coverage for the delayed-free path

## Root Cause

Some KV connectors can finish a request while delaying block free until async transfer cleanup reports completion. After finished_req_ids is cleared, the scheduler could still retain the finished request but report no finished work left. That let the engine stop polling no-forward steps, so connector completion was not observed and KV blocks could remain allocated.
