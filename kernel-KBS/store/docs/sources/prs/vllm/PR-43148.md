## Summary

Centralizes deprecation notices in `vllm/envs.py` via a new `deprecated_env` helper that wraps a getter and emits a `FutureWarning` (visible to end users by default — addressing the gemini-code-assist review on the first iteration) on first read when the var is explicitly set in the environment. Removal target **v0.23**.

### Deprecated env vars

**MoE side** — covered by `--moe-backend` (#33807) and `--quantization_config.moe.activation` (#41566):

| Env var | Replacement |
|---|---|

| `VLLM_USE_FLASHINFER_MOE_FP4` | `--moe-backend flashinfer_{trtllm,cutlass,cutedsl}` |
| `VLLM_USE_FLASHINFER_MOE_FP16` | `--moe-backend flashinfer_trtllm` / `flashinfer_cutlass` |
| `VLLM_FLASHINFER_MOE_BACKEND` | Pass `--moe-backend flashinfer_{trtllm,cutlass,cutedsl}` directly |
