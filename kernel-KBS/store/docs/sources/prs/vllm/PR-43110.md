## Summary

## Purpose
Replace `torch_nccl` as the default EPLB communicator for synchronous mode. Batched isend/irecv with the NCCL backend hangs under high load (many ops, large tensors) — see https://github.com/pytorch/pytorch/issues/174288.

The new default prefers nixl when the package is installed, and falls back to torch_gloo otherwise. A lightweight is_nixl_available() utility is added to vllm/distributed/nixl_utils.py using importlib.util.find_spec so the availability check doesn't trigger any import side effects or log noise.

Fix https://github.com/vllm-project/vllm/issues/38986

## Test Plan
