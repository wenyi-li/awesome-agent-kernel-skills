## Summary

## Purpose

This PR implements two small improvements to the mHC post tilelang kernel

#### 1. Removes output STS instructions

The output is currently written to `x_shared`, then from `x_shared` to HBM. Given that data is never read from `x_shared` later on, this write is unnecessary and is removed here.

#### 2. Vectorized loads/stores

A look at the generated PTX showed that individual 16 bit load/store ops were used for each element. Switching the inner loop to `T.vectorized` enabled 128 bit vectorized load/store ops.
