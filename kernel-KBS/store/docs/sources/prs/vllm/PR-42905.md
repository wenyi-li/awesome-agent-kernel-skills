## Summary

`ModelConfig.renderer_num_workers` sizes the renderer thread pool, but the pool is only consumed by the async renderer path (`_process_multimodal_async`, `render_chat_async`). The offline `LLM` entrypoint calls the synchronous `render_chat` / `_process_multimodal` directly, so `renderer_num_workers > 1` is silently ignored for offline batches.

The docstring also currently claims the pool "handles ... multimodal preprocessing" with no caveat about the sync path — which is misleading for `LLM` users (see [#42901](https://github.com/vllm-project/vllm/issues/42901)).

This PR:
- Clarifies the `renderer_num_workers` docstring in `ModelConfig` to scope it to the async renderer path.
- Emits a one-time `logger.warning_once` in `LLM.__init__` when `renderer_num_workers > 1`, so the no-op is visible.
- Adds a regression test in `tests/entrypoints/llm/test_renderer_workers_warning.py` (~40 lines) covering both the > 1 warning case and the default-silent case.
