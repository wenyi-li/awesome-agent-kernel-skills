## Summary

Two tightly-coupled additions for ModelOpt NVFP4 W4A16:

1. **Fused MoE W4A16 routing** — `ModelOptNvFp4FusedMoE` honors `W4A16_NVFP4` on-disk checkpoints by passing `activation_key=None` to `select_nvfp4_moe_backend`, which filters every W4A4 backend (their `_supports_quant_scheme` requires `(kNvfp4Static, kNvfp4Dynamic)` exactly) and leaves Marlin (which only checks `weight_key`).

2. **Mixed-precision dispatch for W4A16_NVFP4 per-layer entries** — `ModelOptMixedPrecisionConfig.get_quant_method` now recognizes `quant_algo: "W4A16_NVFP4"` entries in `quantized_layers` and routes them to `ModelOptNvFp4W4A16LinearMethod` (Linear) and `ModelOptNvFp4FusedMoE` with `use_a16=True` (MoE) via a sibling `w4a16_nvfp4_config` sub-config built in `_from_config`.


**Tested**: 5 parametrized unit-test cases pass on CPU in ~2s. End-to-end smoke verified on `nvidia/Qwen3.6-35B-A3B-NVFP4` (Qwen3.5-MoE-VL hybrid).
