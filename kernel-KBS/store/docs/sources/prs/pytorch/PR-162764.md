## Summary
Fixes excessive warning output in distributed training by making `TORCH_CUDA_ARCH_LIST` detection messages rank-0 only, while demonstrating that Blackwell (sm100/10.0+PTX) is now the default architecture auto-detected on B200 GPUs.

## Problem
When `TORCH_CUDA_ARCH_LIST` is not explicitly set, PyTorch's cpp_extension module auto-detects visible GPU architectures and logs a warning. In distributed training with `torchrun`, this warning was emitted on every rank, causing excessive log spam. On B200/Blackwell systems, the auto-detected default becomes `TORCH_CUDA_ARCH_LIST='10.0+PTX'`, and this message appeared N times for N processes.

## Solution / Techniques
The fix moves the architecture detection log from a `warning` to a `debug` level message and restricts it to non-distributed execution or rank 0 only. The message is also deferred until after the actual architecture list is computed, so it can display the resolved value (e.g., `10.0+PTX` on Blackwell systems) rather than a generic reminder.

## Key Code
```python
# torch/utils/cpp_extension.py - _get_cuda_arch_flags()
# Before: unconditional warning before arch computation
if not _arch_list:
    logger.warning(
        "TORCH_CUDA_ARCH_LIST is not set, all archs for visible "
        "cards are included for compilation.")

# After: rank-0 only debug message with resolved arch list
if not _arch_list:
    if (not torch.distributed.is_available() or
        not torch.distributed.is_initialized() or
        torch.distributed.get_rank() == 0):
        arch_list_str = ';'.join(arch_list)
        logger.debug(
            "TORCH_CUDA_ARCH_LIST is not set, using "
            "TORCH_CUDA_ARCH_LIST='%s' for visible GPU "
            "architectures.", arch_list_str)
```

## Performance
No kernel-level performance impact. Reduces log noise in distributed training on multi-GPU Blackwell systems. The resolved architecture list on B200 systems shows `10.0+PTX`, confirming that cpp_extension auto-detection correctly identifies Blackwell compute capability for JIT-compiled CUDA extensions.
