## Summary
Adds runtime warnings when PyTorch detects users running on GPU architectures no longer supported by their CUDA build version, implementing a CUDA architecture compatibility matrix that includes sm100 (Blackwell) as a supported target for CUDA 12.8+ and 13.0 builds.

## Problem
With CUDA 12.8+ and 13.0, NVIDIA dropped support for older Maxwell (sm50) and Pascal (sm60) GPU architectures. Users running PyTorch built with these newer CUDA versions on older GPUs would encounter silent failures or cryptic errors. There was no user-facing guidance about which CUDA version to install for their specific GPU architecture, and no awareness that sm100 (Blackwell) requires CUDA 12.8 or newer.

## Solution / Techniques
Implements a `CUDA_ARCHES_SUPPORTED` dictionary in `torch/cuda/__init__.py` that maps CUDA versions to their supported architecture ranges. During `torch.cuda.init()`, the code checks the current GPU's compute capability against the supported range for the installed CUDA version. If the GPU falls outside the range, a `UserWarning` is emitted with remediation guidance. The architecture range for CUDA 12.8/12.9 extends up to sm120, and CUDA 13.0 supports sm75 through sm120, explicitly covering Blackwell (sm100).

## Key Code
```python
# torch/cuda/__init__.py
CUDA_ARCHES_SUPPORTED = {
    "12.6": {"min": 50, "max": 90},
    "12.8": {"min": 70, "max": 120},
    "12.9": {"min": 70, "max": 120},
    "13.0": {"min": 75, "max": 120},
}
```

## Performance
No direct performance impact. This is a user-experience improvement that provides clear guidance when architecture mismatches are detected, preventing users from running unsupported configurations. The warning directs users to install the appropriate CUDA build version for their GPU, including pointing Blackwell (sm100) users toward CUDA 12.8+ or 13.0 builds.
