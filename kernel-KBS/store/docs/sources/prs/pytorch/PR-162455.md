## Summary
Updates PyTorch's CUDA build architecture lists to remove legacy sm50-sm70 support from CUDA 12.6/12.8 aarch64 builds and modernizes the CUDA 13.0 build infrastructure, which targets sm75 through sm120 including Blackwell (sm100) as a first-class architecture.

## Problem
PyTorch's aarch64 (ARM) CUDA builds included architecture support for sm50 through sm70, but these older architectures had no actual ARM platform support. Additionally, the CUDA 13.0 build scripts needed updates for library path resolution and version detection. The x86 build configuration already targeted `TORCH_CUDA_ARCH_LIST="7.5;8.0;8.6;9.0;10.0;12.0+PTX"` for CUDA 13.0, but the aarch64 builds were misaligned.

## Solution / Techniques
Three areas of change:
1. **aarch64 architecture cleanup**: CUDA 12.6 arch list narrowed from `"5.0;6.0;7.0;8.0;9.0"` to `"8.0;9.0"`. CUDA 12.8 narrowed from `"7.0;8.0;9.0;10.0;12.0"` to `"8.0;9.0;10.0;12.0"`, maintaining sm100 (10.0) and sm120 (12.0) Blackwell support.
2. **CUDA 13 version detection**: Updated version string matching from `"130"` to `"13"` for robustness against minor version changes.
3. **Platform constraints removal**: Eliminated `platform_machine == 'x86_64'` restrictions from NVIDIA package dependencies, enabling aarch64 builds to use the same packages.

## Key Code
```bash
# .ci/aarch64_linux/aarch64_ci_build.sh
# CUDA 12.6 - removed sm50-sm70 (no ARM support)
TORCH_CUDA_ARCH_LIST="8.0;9.0"

# CUDA 12.8 - removed sm70, kept sm100 (Blackwell)
TORCH_CUDA_ARCH_LIST="8.0;9.0;10.0;12.0"

# CUDA 13.0 x86 builds (in build_cuda.sh) target:
# TORCH_CUDA_ARCH_LIST="7.5;8.0;8.6;9.0;10.0;12.0+PTX"
```

## Performance
No direct kernel performance impact. This is a build infrastructure change that ensures Blackwell (sm100/10.0) and sm120 (12.0) are properly included in the CUDA 12.8 and 13.0 architecture target lists for both x86 and aarch64 platforms, enabling optimized code generation for these architectures.
