## Summary
Updates the CUDA 13.0 architecture support range in PyTorch's runtime compatibility checks, adjusting the minimum supported compute capability from sm70 to sm75, aligning with the actual CUDA 13.0 build configuration that targets `TORCH_CUDA_ARCH_LIST="7.5;8.0;8.6;9.0;10.0;12.0+PTX"`.

## Problem
The initial `CUDA_ARCHES_SUPPORTED` dictionary in PyTorch set CUDA 13.0's minimum supported architecture to sm70. However, the actual CUDA 13.0 build configuration (in `.ci/manywheel/build_cuda.sh`) specifies `TORCH_CUDA_ARCH_LIST="7.5;8.0;8.6;9.0;10.0;12.0+PTX"`, which starts at sm75 (Turing). This meant the warning system was not correctly identifying sm70 (Volta) GPUs as unsupported when running CUDA 13.0 builds. The architecture list explicitly includes 10.0 (Blackwell/sm100) and 12.0 (sm120) as supported targets.

## Solution / Techniques
A one-line change to the `CUDA_ARCHES_SUPPORTED` dictionary updates the CUDA 13.0 entry's minimum from 70 to 75, correctly reflecting that Volta (sm70) is not included in the CUDA 13.0 build's architecture list. This ensures users on Volta GPUs receive appropriate warnings directing them to install CUDA 12.8 or 12.9 builds instead.

## Key Code
```python
# torch/cuda/__init__.py - updated CUDA 13.0 entry
CUDA_ARCHES_SUPPORTED = {
    "12.6": {"min": 50, "max": 90},
    "12.8": {"min": 70, "max": 120},
    "12.9": {"min": 70, "max": 120},
    "13.0": {"min": 75, "max": 120},  # changed from min: 70 to min: 75
}
```

## Performance
No direct performance impact. This is a correctness fix for the architecture compatibility warning system. The supported architecture list for CUDA 13.0 builds confirms that sm100 (Blackwell) is a first-class target at `10.0` in the arch list, alongside sm90 (Hopper) at `9.0` and sm120 at `12.0+PTX`.
