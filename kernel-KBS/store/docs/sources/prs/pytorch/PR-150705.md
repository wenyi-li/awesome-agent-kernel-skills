## Summary
Restricts 128-bit vectorized memory operations (vec128/vec8) to CUDA 12.8+ toolkits only, ensuring correct behavior on Blackwell-class GPUs while maintaining backward compatibility with older CUDA versions and ROCm.

## Problem
PyTorch introduced 128-bit (16-byte) vectorized loads and stores for CUDA kernels to improve memory throughput. However, these vec128 operations caused issues when compiled with CUDA toolkits older than 12.8. The CUDA 12.8 toolkit (required for Blackwell/sm100 support) properly handles vec8 alignment and vectorized access patterns, but older toolkit versions do not. This created build and runtime failures when the same codebase was compiled across different CUDA versions.

## Solution / Techniques
The fix gates vec128 vectorization behind CUDA version preprocessor checks. When `CUDA_VERSION >= 12080`, the full vec8 (128-bit) vectorization path is enabled. For older CUDA versions, the code falls back to vec4 (64-bit) vectorization. Key changes span multiple files:

- `MemoryAccess.cuh`: Guards `vec8_alignment` behind version check
- `CUDALoops.cuh`: Conditionally enables vec8 kernel dispatch
- `Dropout.cu`: Adds version-branched vectorization for dropout kernels
- `thread_constants.h`: Sets `thread_work_size()` to 8 for CUDA 12.8+, 4 otherwise

## Key Code
```cuda
// aten/src/ATen/native/cuda/MemoryAccess.cuh
#if defined(USE_ROCM) || (defined(CUDA_VERSION) && CUDA_VERSION >= 12080)
  constexpr int vec8_alignment = 16; // 128-bit
#endif

// aten/src/ATen/native/cuda/thread_constants.h
#if defined(CUDA_VERSION) && CUDA_VERSION < 12080
  constexpr int thread_work_size() { return 4; }
#else
  constexpr int thread_work_size() { return 8; }
#endif
```

## Performance
The vec128 optimization doubles the effective memory bandwidth utilization for elementwise kernels by loading 16 bytes per thread instead of 8. This is particularly beneficial on Blackwell GPUs where the memory subsystem is designed to handle wider vector loads efficiently. The gating ensures no regressions occur on older CUDA toolkit versions that lack proper vec128 support.
