## Summary

Fixes critical bugs in the CuTe DSL implementation of SM100 block-scaled GEMM kernels when using overlapping accumulators. The overlapping accumulator technique double-buffers TMEM accumulators so that the epilogue-fusion can drain one accumulator while the MMA fills the next, but incorrect buffer indexing and early-release logic caused data corruption on Blackwell GPUs.

## Problem

When running SM100 block-scaled GEMM with overlapping accumulators (where MMA and epilogue-fusion overlap execution via double-buffered TMEM), two bugs caused incorrect results:

1. **Buffer index collision**: The epilogue-fusion used `real_subtile_idx` for shared memory c-buffer indexing, but this index did not account for the overlap pipeline state, causing two execution phases to collide on the same buffer.

2. **Early release miscalculation**: The logic computing `iter_acc_early_release_in_epilogue-fusion` (when to signal that an accumulator can be reused) used integer division instead of ceiling division, causing incorrect release timing when scale factor TMEM columns were not evenly divisible by the epilogue-fusion tile width.

The author demonstrated the failure with a specific test case: `mnkl: (16384, 2048, 512, 1)` with MMA tiler (256, 256) and cluster shape (2, 1), showing ~1.5% of tensor elements were mismatched with errors exceeding tolerance.

## Solution / Techniques

Three targeted fixes in the CuTe DSL blockscaled GEMM persistent kernels:

1. **Buffer indexing**: Changed from `real_subtile_idx` to `subtile_idx` for c-buffer stage selection, ensuring proper round-robin buffer assignment across pipeline stages.

2. **Early release calculation**: Replaced integer division with ceiling division minus one:
   ```python
   # Before (incorrect)
   self.iter_acc_early_release_in_epilogue-fusion = self.num_sf_tmem_cols // self.epi_tile_n
   
   # After (correct)
   num_subtiles_in_overlap_region = ceil_div(self.num_sf_tmem_cols, self.epi_tile_n)
   self.iter_acc_early_release_in_epilogue-fusion = num_subtiles_in_overlap_region - 1
   ```

3. **Thread configuration**: Replaced hardcoded warp size with `self.threads_per_warp` for barrier thread counts, ensuring correct synchronization.

## Key Code

```python
# Fixed c-buffer stage selection in epilogue-fusion loop
# dense_blockscaled_gemm_persistent.py

# Before (causes buffer collision in overlap mode)
c_buffer = (num_prev_subtiles + real_subtile_idx) % self.num_c_stage

# After (correct buffer rotation)
c_buffer = (num_prev_subtiles + subtile_idx) % self.num_c_stage
```

```python
# Fixed accumulator early release timing
def ceil_div(a, b):
    return (a + b - 1) // b

num_subtiles_in_overlap_region = ceil_div(self.num_sf_tmem_cols, self.epi_tile_n)
self.iter_acc_early_release_in_epilogue-fusion = num_subtiles_in_overlap_region - 1
```

## Performance

The fix restores correctness for overlapping accumulator mode without impacting performance. The overlapping accumulator optimization itself is critical for SM100 performance: by allowing the epilogue-fusion to drain results from one TMEM accumulator bank while the MMA fills another, it hides epilogue-fusion latency and improves overall GEMM throughput. All changes were validated on a B200 GPU.
