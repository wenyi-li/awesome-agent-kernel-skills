## Summary

Adds support for zero-K dimension handling in Blackwell SM100 grouped GEMM kernels. When a group in a grouped GEMM has K=0 (no contraction dimension), the kernel must correctly skip the MMA computation and produce zero-filled output instead of hanging or producing undefined results.

## Problem

Grouped GEMM operations allow batching multiple GEMMs with different problem sizes into a single kernel launch. In production workloads (e.g., Mixture-of-Experts models), some groups may have K=0 when an expert receives no tokens. The existing SM100 grouped GEMM kernels did not handle this edge case: the accumulator pipeline would deadlock waiting for MMA results that were never produced, causing the kernel to hang. This was reported as issue #2674.

## Solution / Techniques

The fix introduces `is_accumulator_needed = K > 0` guards throughout the SM100 grouped GEMM pipeline:

1. **Epilogue Files**: Conditional accumulator handling in both SMEM and no-SMEM epilogue-fusion variants. When K=0, registers are zero-filled directly instead of waiting for accumulator pipeline results.

2. **Collective MMA**: The pipeline acquire in the blockscaled MMA array is wrapped in `if (k_tile_count > 0)`, preventing deadlock when there are no tiles to process.

3. **Kernel Orchestration**: Multiple guards in the top-level kernel:
   - MMA execution gated by `if (is_mma_leader_cta && k_tile_count > 0)`
   - Pipeline state increments skipped when K=0
   - Wave increment gated by `if (size<2>(problem_shape_MNKL) > 0)`

4. **Test Updates**: Random problem generation in examples 75 (grouped GEMM) and block_scaled tests updated to allow K=0 as a valid dimension.

## Key Code

```cpp
// Kernel-level K=0 guard (sm100_gemm_array_tma_warpspecialized.hpp)
bool is_accumulator_needed = K > 0;

if (is_mma_leader_cta && k_tile_count > 0) {
    // Execute MMA pipeline
    collective_mainloop.mma(...);
    // Increment pipeline state
}

// Epilogue K=0 handling (sm100_epilogue-fusion_array_nosmem.hpp)
if (is_accumulator_needed) {
    // Wait for accumulator from MMA pipeline
    acc_pipeline.consumer_wait(acc_consumer_state);
} else {
    // Zero-fill accumulators directly
    fill(accumulators, ElementAccumulator(0));
}
```

## Performance

No performance regression for the normal K>0 path, as the guards are simple branch conditions evaluated once per group. The fix enables correct execution of MoE workloads where expert routing may produce empty groups, eliminating kernel hangs that previously required workarounds.
