## Summary

> This PR would not have been possible without the pioneering work by @eraser00 in #18246, who identified the think_end token missing problem and built the first strict reasoning grammar to address it. Their implementation and insights laid the groundwork for everything here — we took that foundation, refactored and extended it into the two-phase constrained decoding system below. Thank you!

## Summary

Refactor `ReasonerGrammarBackend` into a `THINKING → GENERATION` state machine that wraps any grammar backend. Grammar constraints only activate after the reasoning phase ends, so reasoning models can produce `<think>...</think>` freely while still emitting valid JSON / regex / EBNF afterwards.

```bash
sglang serve --model Qwen/Qwen3-235B-A22B \
  --reasoning-parser qwen3 --enable-strict-thinking
```

### What's new

- **Two-phase `ReasonerGrammarObject`** with separate thinking/generation token counters; rollback-safe for speculative decoding.
- **`--enable-strict-thinking` server flag**: during the thinking phase, a GPU bitmask filters out per-detector excluded tokens (e.g. tool-call markers, EOS variants) so the model cannot emit them inside `<think>`. Triton kernel + Torch CPU
