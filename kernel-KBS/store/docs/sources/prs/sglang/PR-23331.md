## Summary of Issues
### `AttributeError: 'RadixLinearAttention' object has no attribute 'is_cross_attention'`
The MTP draft model mutates `hf_config` during construction, but `get_config()` is cached by `lru_cache`, so the draft and target models share the same object. This pollutes the target's `full_attention_layer_ids`, causing linear-attn layers to be misrouted to the full-attn backend. **Fix:** `deepcopy(config)` in the MTP constructor to isolate the mutation; additionally, rewrite `_is_full_attn` to dispatch via `isinstance(layer, RadixLinearAttention)` so routing no longer depends on the easily-polluted ID list.
