## Motivation

The aiter attention backend crashes on ROCm with
`HSA_STATUS_ERROR_MEMORY_APERTURE_VIOLATION` /
`HIP error: an illegal memory access was encountered` once the running
batch's `seq_lens_sum` grows past the cuda-graph KV-indices buffer.
The crash typically lands at a downstream `torch.cat` /
`synchronize()` so it looks unrelated, but the actual fault is an OOB
`tl.store` inside `create_flashinfer_kv_indices_triton`.

Trigger conditions (must all hold):
- `--attention-backend aiter`, non-MLA model, `--page-size > 1`
- CUDA graph enabled (default)
- `seq_lens_sum` at decode time exceeds
  `cuda_graph_max_bs * ceil(max_context_len / page_size)` — easy to
  hit at moderate concurrency on long-context models, especially when
  `cuda_graph_max_bs` is small (e.g. clamped by
  `--max-running-requests`).

at page granularity (`max_bs * ceil(max_context_len / page_size)`),
but every forward path that fills it (default decode, MLA decode,
target_verify, draft_extend{,_v2}) writes per token via
