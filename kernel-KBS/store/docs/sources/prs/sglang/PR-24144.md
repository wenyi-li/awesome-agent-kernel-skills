## Summary

## Motivation

When running Qwen3.5-VL models with speculative decoding (`--speculative-algo NEXTN`), the model architecture name is transformed from `Qwen3_5ForConditionalGeneration` to `Qwen3_5ForCausalLMMTP` (Multi-Token Prediction draft model). However, `Qwen3_5ForCausalLMMTP` is not registered in `QwenVLImageProcessor.models`, causing the multimodal processor lookup to fail with:
```bash
  ValueError: No processor registered for architecture: ['Qwen3_5ForCausalLMMTP'].
```

Additionally, there is a duplicated debug log (Get embedding slice for {modality}) in the multimodal embedding processing path that adds unnecessary noise.

## Modifications

1. Added `Qwen3_5ForCausalLMMTP` to the multimodal processor registration in `python/sglang/srt/multimodal/processors/qwen_vl.py`
2. Remove duplicated log: Removed the redundant `logger.info(f"Get embedding slice for {modality}, num_tokens={num_tokens}")` call in the embedding slicing logic.
