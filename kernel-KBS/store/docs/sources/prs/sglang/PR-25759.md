## Summary

## Motivation

Launching an encoder-only EPD server with `Qwen3VLMoeForConditionalGeneration` crashes during weight loading:

  File "python/sglang/srt/models/qwen3_vl_moe.py", line 238, in load_weights
      and hasattr(self.model, "start_layer")
  AttributeError: 'Qwen3VLMoeForConditionalGeneration' object has no attribute 'model'

  In `encoder_only` mode the parent class skips creating `self.model` (qwen3_vl.py:1123-1148), but `load_weights` evaluates `self.model` directly — `hasattr` does not catch the AttributeError raised by the first argument.

  The parent class `Qwen3VLForConditionalGeneration.load_weights` already guards with `hasattr(self, "model")` (qwen3_vl.py:1345); the MoE subclass copied the older pattern in #19135 and lost that guard.
