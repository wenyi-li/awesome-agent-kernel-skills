## Summary

Add extension points to `OpenAIServingChat` that allow subclasses to customize chat completion behavior without modifying core code. These hooks enable downstream implementations to integrate custom rendering libraries while maintaining compatibility with upstream SGLang.

## Motivation

Server-side rendering libraries like Prime Intellect's [renderers](https://github.com/PrimeIntellect-ai/renderers) provide programmable chat templates that can:

1. **Encode messages → token IDs** with precise control over special tokens and message boundaries
2. **Parse completion IDs → structured responses** (content, reasoning_content, tool_calls) by scanning for special-token boundaries
3. **Bridge multi-turn rollouts** without re-rendering history (critical for RL training where token identity must be preserved)

From the renderers README:
> For RL the trainer must see the exact token ids the sampler saw. The standard alternative — let the inference engine apply the chat template, parse tool calls, parse reasoning, and re-render full history every turn — silently breaks token identity.
