## Summary

Consolidate `batch.seq_lens_sum` / `seq_lens_cpu` maintenance into a single helper `ScheduleBatch.refresh_seq_lens_cpu(sync: bool = True)`. Producers no longer compute the sum inline; the helper is the unique compute site.

### Changes

- **`schedule_batch.py`**: new `refresh_seq_lens_cpu(sync)` method. `sync=True` D2Hs from `seq_lens` (needed for spec v2's mid-forward GPU rebind); `sync=False` only recomputes the cached sum from the current `seq_lens_cpu`. `filter_batch` / `merge_batch` / `prepare_for_decode` now leave `seq_lens_sum = None` (deferral marker) instead of inline `+= / .sum()`.
- **`scheduler.py`**: single pre-forward `batch.refresh_seq_lens_cpu()` call right BEFORE `_overlap_forward_isolation` (placement matters — inside the isolation block would get reverted by snapshot/restore).
- **`forward_batch_info.py`**: `init_new` lazy-refreshes via helper when `batch.seq_lens_sum is None` (covers non-overlap forward paths).
