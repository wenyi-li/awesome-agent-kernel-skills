## Summary

## Motivation
Fixes #24637.
`test/registered/mla/test_flashmla.py` currently forces the FlashMLA MTP speculative decoding test to fall back to spec decoding v1 by setting `SGLANG_ENABLE_SPEC_V2=False`. FlashMLA should support spec decoding v2 and the registered CUDA CI test should cover that path explicitly.
## Modifications
- Enable spec decoding v2 in `TestFlashMLAMTP` by changing `SGLANG_ENABLE_SPEC_V2.override(False)` to `SGLANG_ENABLE_SPEC_V2.override(True)`.
- Treat `ForwardMode.DRAFT_EXTEND_V2` as a speculative draft-extend mode in DeepSeek MLA attention backend dispatch.
- Route FlashMLA `DRAFT_EXTEND_V2` forward calls through the extend/prefill path instead of the target-verify FlashMLA path, matching the existing metadata initialization behavior.
