## Summary

Replace the in-file SM90 FP8 W8A8 GEMM dispatch in `fp8_gemm_kernel.cu` with a modular swap-AB dispatch in `cutlass_extensions/`. Adds two new dispatch buckets (`M32_N8192` and `M128_smallN`) that close the regression zones a naive swap-AB dispatch would leave behind.

Net result on H200:
- **1.156x geomean** vs main on M ≤ 128 kernel-level (58W / 4L / 84 shapes), no net regression in any M bucket
- **+5.8 – 18.5% end-to-end decode throughput** on Llama-3.1 8B/70B FP8-dynamic

## Motivation

Main's SM90 FP8 dispatch uses tile `<_64, _64, _128>` for all small-M paths. For M_orig ≪ 64 (decode batch=1..16) this wastes a large fraction of the M-tile every iteration — e.g. for M=1 only 1/64 of the M dim is real work, the rest is padding-compute.

Swap-AB transposes the GEMM at the kernel level (D^T = B^T · A^T) so that "kernel M" maps to the much larger N_orig dimension and "kernel N" maps to the small M_orig. Pairing this with smaller kernel-N tiles (16, 32) lets us sit just above M_orig with minimal padding waste, which is the dominant overhead in small-batch decode.
