## Summary

## Motivation

When the pipeline executes test cases, too many logs are printed, affecting the execution time and locating the failure cause of failed test cases.
The root cause of this problem is that the transformer version is updated. As a result, a compatibility warning is reported when the related method is called.

## Modifications

An environment variable for controlling the transformer alarm severity is added to the test case. Only errors of the error level are printed in logs.
