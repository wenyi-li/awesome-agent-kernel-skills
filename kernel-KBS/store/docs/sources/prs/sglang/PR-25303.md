## Summary

## Motivation

Fixes #22101.

The standalone spec-v1 Triton GSM8K CI test is sensitive to small numerical differences in the target-verify attention path at greedy temperature 0. The test is intended to cover standalone speculative decoding, but native Triton target verify can move near-threshold runs across the existing accuracy cutoff. This PR makes the Triton standalone spec test run with SGLang's existing deterministic inference mode.

## Findings

The target-verify boundary was checked before changing the test:

- draft-tree mask construction
- target positions
- KV slot layout and `out_cache_loc`
- accepted-token transitions
- bonus-token handling
- target-vs-reference logits

Those checks did not show scheduler or verifier-state corruption. The remaining observed difference was numerical: the native Triton target-verify path can produce small logit drift around near-tie tokens, which is enough to flip greedy argmax and change the accepted speculative path.
