## Summary

The lazy SGLANG_PREP_IN_CUDA_GRAPH upgrade runs real GPU kernels. When fired from inside MQALayer._forward_prepare_multi_stream, it lands on whichever alt-stream reaches it first (stream_indexer via forward_indexer_compressor, or stream_compressor via forward_core_compressor), and the other alt-stream then performs an unsynchronized cross-stream read of the freshly allocated metadata tensors. PP=2 happens to hide this (1.6T layer 0/1 are C=128 so only stream_compressor needs the upgrade); PP>=4 middle ranks own only middle layers where the first multi-stream layer can be C=4 and the race fires.

Run the upgrade on the main stream before any alt-stream fork so all forks see the upgrade through their existing wait_stream(main). Subsequent in-alt-stream calls are no-ops.
