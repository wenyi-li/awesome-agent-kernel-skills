## Summary

## Motivation
                                                                                                                                                                         
  The Mooncake, Mori, and Nixl disaggregation backends share significant duplicated logic for heartbeat checking, node failure handling, KV transfer preparation, and timeout management. This duplication increases maintenance burden and makes it easy for bug fixes in one backend to be missed in others.

## Modifications

### Consolidate shared logic into the common base classes (CommonKVManager, CommonKVSender, CommonKVReceiver)

#### Common Manager (common/conn.py):
  - _start_heartbeat_checker_thread(): unified heartbeat logic with _on_heartbeat_success() hook for backend-specific cleanup
  - _handle_node_failure(): shared failure propagation via record_failure + update_status

#### Common Sender (common/conn.py):
  - _prepare_send(): shared index tracking and CP-rank filtering
  - _check_bootstrap_timeout(): shared bootstrap timeout logic
  - pop_decode_prefix_len() / should_send_kv_chunk(): moved default implementations to base

#### Common Receiver (common/conn.py):
  - _check_wa
