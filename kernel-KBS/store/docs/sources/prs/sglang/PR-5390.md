## Summary

Adds support for the Blackwell CUTLASS MLA (Multi-Head Latent Attention) decode kernel to SGLang, enabling hardware-accelerated attention for DeepSeek models on SM100 GPUs. Introduces a new `tcgen05_mla` attention backend option that leverages specialized Blackwell tensor core kernels for MLA decode.

## Problem

Prior to this change, SGLang only supported Triton-based attention backends for MLA models like DeepSeek-R1. On Blackwell GPUs, the Triton backend did not fully exploit the hardware capabilities of the SM100 architecture, resulting in suboptimal decode throughput and higher latency.

## Solution / Techniques

- Implements a new `tcgen05_mla_backend.py` attention backend wrapping the CUTLASS MLA decode kernel optimized for Blackwell
- Fixed page size of 128 tokens for kernel compatibility
- Requires float16 or bfloat16 precision
- Integrates with the existing radix attention framework and CUDA graph support
- Leverages Blackwell tensor memory and TMA for efficient data movement

## Key Code

```python
# Server launch with Cutlass MLA backend
--attention-backend tcgen05_mla --enable-dp-attention --dtype float16 --page-size 128
```

## Performance

Benchmarked on DeepSeek-R1 with 3,000 requests:

| Metric | Cutlass MLA | Triton Baseline |
|--------|-------------|-----------------|
| Throughput | 10,447 tok/s | 8,227 tok/s |
| Mean E2E Latency | 572.8s | 726.5s |
| Mean TTFT | 134.2ms | 157.2ms |

Approximately 27% throughput improvement and 21% latency reduction compared to the Triton baseline on Blackwell hardware.
