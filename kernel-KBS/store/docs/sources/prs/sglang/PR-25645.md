## Summary

## Motivation

At the moment GLM-Image can generate images only on 1 device, sp>1 has errors with shapes, tp>1 has accuracy issues

## Modifications

`shard_rotary_emb_for_sp` for cos and sin

moved GlmImageAR to a new stage - while other models in `GlmImageBeforeDenoisingStage` implemented in `/multimodal_gen`, GlmImageAR used from transformes and can be implemented in SGLang in future in `/srt`

added new parallelism paradigm `StageParallelismType.MAIN_RANK_ONLY_AND_SEND_TO_OTHERS` to run GlmImageAR. Before this change every rank called GlmImageAR.generate independently and on every rank it produced different tokens.
