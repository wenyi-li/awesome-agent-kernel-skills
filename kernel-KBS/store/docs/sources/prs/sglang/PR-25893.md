## Summary

This avoids rebuilding dynamic LoRA weights when a previously applied adapter is only deactivated and then reactivated with the same single-adapter configuration.

The fast path only toggles the wrapped LoRA layers back on when the cached A/B tensors, path, and strength still match. Merged LoRA, previously merged layers, and layerwise-offloaded modules still use the existing weight-update context.

## Impact

- Reduces dynamic LoRA reactivation from a full per-layer LoRA state rebuild to metadata reactivation when the cached adapter still matches.
- Preserves the existing path for merged LoRA and offloaded components.
- Also handles the API case where `set_lora` passes only an already-loaded nickname and omits `lora_path`.
- Latest high-memory resident LTX2 two-stage mode keeps both DiTs resident and already reports near-zero `ltx2_lora_switch_stage2`, so this PR is primarily justified by dynamic/offload modes rather than that resident path.
