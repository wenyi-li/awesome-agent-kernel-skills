## Summary

Replaces the per-tensor quantized BMM FP8 kernel with DeepGemm's per-token-group quantized `grouped_gemm_masked` kernel for MLA decode in DeepSeek-V3 inference. The finer-grained quantization with masked grouped GEMM consistently outperforms the simpler per-tensor BMM FP8 approach.

## Problem

DeepSeek-V3 decode performance was bottlenecked by the BMM FP8 kernel used for MLA attention logits computation. Per-tensor quantization with standard BMM was suboptimal, consuming 7.9 microseconds for single-batch decode on H200 hardware. A more efficient approach was needed that combined better quantization granularity with faster kernel execution.

## Solution / Techniques

- Replaces BMM FP8 with DeepGemm's `grouped_gemm_masked` using per-token-group quantization
- Per-token-group quantization provides better numerical accuracy while enabling use of the faster masked grouped GEMM kernel
- Combined quantization + zeros kernel takes approximately 2.79 microseconds vs 7.9 microseconds for BMM FP8 on single-batch decode
- Maintains accuracy on GSM8K benchmark (0.933-0.939 range)

## Key Code

```python
# Replace per-tensor quant + gemm_fp8 with:
# per-token-group quant + deep_gemm.grouped_gemm_masked
# The grouped_gemm_masked kernel operates on masked expert groups
# enabling efficient MoE-style batched computation
```

## Performance

On 8xH200 with DeepSeek-V3:

| Metric | Before | After |
|--------|--------|-------|
| 1-batch decode kernel time | 7.9 us | 2.79 us |
| 50-batch decode kernel time | 6.4 us | 4.2 us |
| Median decode throughput | 58.15 tok/s | 60.24 tok/s |

Approximately 3% decode latency reduction with consistent accuracy (GSM8K: 0.933-0.939).
