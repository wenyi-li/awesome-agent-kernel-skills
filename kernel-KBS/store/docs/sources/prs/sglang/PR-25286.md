## Summary

## Motivation

Fixes #25227.

`google/gemma-4-31B-it` failed to load with `--quantization fp8 --tp 2` when the Triton W8A8 FP8 path received an online-quantized weight scale as a row vector. The GEMM was semantically valid, but `triton_scaled_mm` only accepted the same per-output-channel scale as a column vector.

## Modifications

- Normalize scale tensors at the `triton_scaled_mm` boundary so scalar/1D scales become column-shaped, existing column scales are preserved, and row-vector per-output-channel weight scales shaped `(1, N)` become `(N, 1)`.
- Keep the existing scale assertions after normalization so unsupported scale layouts still fail before kernel launch.
- Add a regression case to the existing `test_triton_scaled_mm.py` coverage that calls `triton_scaled_mm` with `scale_b` shaped `(1, N)` and compares it with the reference output.
