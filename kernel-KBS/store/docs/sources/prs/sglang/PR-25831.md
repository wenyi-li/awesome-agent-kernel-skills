## Summary
- Replace `test_srt_backend.py` (frontend DSL smoke) with a 3-kit + hellaswag gate in `test/registered/core/test_basic_sanity.py`
- Slim dsv4 e2e tests to only `BasicDecodeCorrectnessMixin` + `GSM8KMixin` (drops 80 lines of inline GSM8K duplication)
- Reorganize `test/registered/core/` to keep only framework-level tests; relocate misclassified files to `attention/`, `radix_cache/`, `models/`, `unit/managers/`
- Consolidate model-family e2e tests into `test/registered/models_e2e/` (absorbs `dsv4/` + `dsa_models_e2e/` + 8 single-model files from `models/`)

## Changes

**Stage-a gate**
- Add `python/sglang/test/kits/basic_{api_contract,decode_correctness,scheduler_stress}_kit.py` — 3 reusable mixins targeting protocol / output-quality / scheduler-path failure modes
- Add `test/registered/core/test_basic_sanity.py` mixing all 3 kits + a hellaswag-based `test_accuracy_floor` private to the gate
- Delete `test/registered/language/test_srt_backend.py` (covered by the new gate)
- Delete `python/sglang/test/kits/server_sanity_kit.py` (12 probes split across the 3 new kits)
- Delete `test/registered/debug_utils/test_cuda_coredump.py` (already `disabled=`, no callers)
