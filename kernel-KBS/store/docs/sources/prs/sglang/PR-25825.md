## Summary

## Motivation

Several model attention layers check whether the current layer is the first layer on this PP rank using \`forward_batch.token_to_kv_pool.start_layer\`. This is static per-process configuration (determined by \`make_layers()\` at model init time) that does not change between forward passes and has no business being accessed via \`ForwardBatch\` at all.

## Modifications

For each affected model (llama, qwen3, qwen2, glm4_moe, qwen3_moe, qwen2_moe):
- Add \`start_layer: int = 0\` parameter to the attention layer \`__init__\`
- Set \`self.start_layer = start_layer\` in the attention layer \`__init__\`
- Thread \`start_layer\` from model → decoder layer → attention layer at construction time (using existing \`self.start_layer\` already computed by \`make_layers()\`)
- Change \`forward\` to read \`self.start_layer\` instead of \`forward_batch.token_to_kv_pool.start_layer\`

Also fixes positional-argument breakage in EAGLE subclass constructors (`llama_eagle.py`, `llama_eagle3.py`, `qwen2_eagle.py`) that called `super().__init__` with positional args and silently mis-routed `quant_config` → `start_layer` after the new parameter was inserted.

No runtime behavior change — s
