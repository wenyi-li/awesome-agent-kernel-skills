## Summary

## Motivation

At the moment, Flux.2-Klein-Base model is indirectly supported within SGLang via distilled Flux.2-Klein implementation, which however doesn't allow negative prompts and CFG parallelism.
This PR solves the issue by adding Flux.2-Klein-Base configs and fixes required to make its negative prompts work.

## Modifications

* Added FLUX.2-klein-base pipeline config:
   * New `Flux2KleinBasePipelineConfig`, extending `Flux2KleinPipelineConfig` with `should_use_guidance=True` (Klein-base is undistilled and uses guidance embeddings).
   * Implemented `prepare_neg_cond_kwargs` to build `freqs_cis` for negative-prompt embeddings on the CFG path.
* Added FLUX.2-klein-base sampling defaults:
   * New `Flux2KleinBaseSamplingParams`, extending `FluxSamplingParams` with `num_inference_steps=50`, `guidance_scale=4.0`, and an empty default `negative_prompt`.
* Added FLUX.2-klein-base to multimodal registry:
   * Updated the existing Klein detector to exclude `"base"` so distilled and undistilled variants are routed to their respective configs.
* Generalized empty-string negative-prompt handling:
   * In `python/sglang/multimodal_gen/runtime/pipelines_core/stages/text_en
