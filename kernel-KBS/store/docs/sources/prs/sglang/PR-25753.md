## Summary

## Motivation
## close https://github.com/sgl-project/sglang/issues/25752

Verified as an effective fix.
                                                                                                                                                                                 
Hybrid models like Bailing-2.6-Flash use HybridLinearKVPool as token_to_kv_pool (because they have both Mamba/linear layers and full attention layers with MLA). The assertion in    prepare_chunked_prefix_cache_info only accepted MLATokenToKVPool directly, causing the service to crash under long-context scenarios when chunked prefix cache is triggered.         
                                                                                                                                                                                       
However, HybridLinearKVPool with use_mla=True delegates all full-attention KV operations to its internal full_kv_pool (which is an MLATokenToKVPool). The chunked prefix cache logic never accesses token_to_kv_pool after the assertion — it only operates on req_to_token_pool and batch metadata. So the assertion was overly restrictive.

## Problem

feat: support HybridLinearKVPool in chunked prefix cache handling
