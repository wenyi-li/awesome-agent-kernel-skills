## Summary

Integrates a batch matrix multiplication (BMM) FP8 kernel into the gemm package. This enables efficient reduced-precision matrix multiplication for MLA attention computation in DeepSeek models, leveraging FP8 tensor cores on Hopper and Blackwell GPUs.

## Problem

SGLang needed a dedicated FP8 BMM kernel within its gemm package to support efficient MLA decode operations for DeepSeek models. Without this, the framework relied on higher-precision operations that underutilized the FP8 tensor core capabilities of modern NVIDIA GPUs (SM90+).

## Solution / Techniques

- Adds a native BMM FP8 kernel to gemm as a CUDA extension
- Supports per-tensor FP8 scaling factors for quantized matrix multiplication
- Targets SM89+ architectures (Ada Lovelace, Hopper, Blackwell)
- Integrates with PyTorch's custom ops framework for seamless Python interop
- Validated with comprehensive test suite (6 passed, 2 skipped)

## Key Code

```python
# BMM FP8 kernel integration into gemm
# Supports batch matrix multiplication with FP8 quantization
# for MLA attention logits computation
import sgl_kernel
output = sgl_kernel.gemm_fp8(
    input_a, input_b,
    scale_a, scale_b,
    out_dtype=torch.bfloat16
)
```

## Performance

The FP8 BMM kernel provides reduced memory bandwidth requirements compared to BF16/FP16 BMM operations, enabling higher throughput for MLA decode. Exact speedup depends on matrix dimensions and batch sizes. Test suite confirms correctness with 6/8 tests passed (2 skipped for unsupported configurations).
