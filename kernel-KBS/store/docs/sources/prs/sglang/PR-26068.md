## Summary

Adds a new Claude Code skill at `.claude/skills/sglang-cherrypick/SKILL.md`. Given a release branch and a list of merged PRs, the skill dispatches `.github/workflows/bot-cherry-pick.yml` for each, monitors the resulting workflow runs to completion, and reports per-PR success/failure with links to the created cherry-pick PRs.

### Pre-flight (runs before any workflow dispatch)

- **Validates each PR** is `MERGED` with a recorded merge commit. Fails the whole batch up front rather than triggering partial dispatches.
- **Indexes existing cherry-pick PRs** on the target branch (`gh pr list --base <branch> --label cherry-pick --state all`) and classifies each input PR as already-merged / open-duplicate / closed-only / new. Skips the dispatch when a cherry-pick already landed or is in flight, so we don't create redundant PRs.
- **Simulates the cherry-pick locally** with `git merge-tree --write-tree --merge-base=<sha>^ <target> <sha>`. This is side-effect-free (no working-tree change, no ref update) and matches the workflow's `-m 1` semantics for merge commits.
