## Summary

Integrates a blockwise FP8 quantization kernel into SGLang to optimize DeepSeek-V3 inference. Uses 128x128 block-scale FP8 quantization with CUTLASS kernels, matching the native quantization format used by DeepSeek-V3 weights and achieving significant latency improvements.

## Problem

DeepSeek-V3 uses a block-wise FP8 quantization scheme with 128x128 blocks. The existing per-tensor FP8 quantization in SGLang did not match this native format, requiring suboptimal conversion or falling back to higher-precision computation. A dedicated blockwise FP8 kernel was needed to fully exploit the model's native quantization.

## Solution / Techniques

- Integrates CUTLASS-based blockwise FP8 GEMM kernel matching DeepSeek-V3's 128x128 block quantization format
- Fine-grained per-block scaling factors provide better numerical accuracy than per-tensor quantization
- Built on prior work from PR #3267
- Validated accuracy on GSM8K and MMLU benchmarks
