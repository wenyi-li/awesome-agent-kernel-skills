## Summary

## Motivation

Fixes #25267

`bench_serving --disable-stream` crashed for reasoning models because the non-streaming OpenAI chat path copied only `choices[0].message.content` into `RequestFuncOutput.generated_text`. With `separate_reasoning` enabled, the server can split generated tokens across `message.reasoning_content` and `message.content`. If the model spends the whole completion budget in reasoning, `content` is `null` while `reasoning_content` contains the generated text. That left `generated_text=None`, and `calculate_metrics()` later passed it to `tokenizer.encode()`, which requires a string.

## Modifications

- Add a shared OpenAI chat text combiner for benchmark responses that reconstructs generated text as `reasoning_content + content`, preserving the same ordering already used by the streaming reasoning-content fix.
- Use that combiner in the non-streaming chat-completions path before metrics retokenization.
- Reuse it in the streaming delta path so streaming and non-streaming chat response parsing share the same reasoning/content reconstruction rule.
- Add regression coverage for non-streaming reasoning-only, mixed reasoning/content, and content-only chat
