## Summary

## Motivation

Under the TITO path in RL, we want to preserve model-generated messages as much as it originally is, which means we should avoid doing modifications to whatever the model generates.

This issue was found while validating token-in-token-out (TITO) support for DeepSeek-V4-Flash. During testing, DeepSeek-style reasoning output such as:

`<think>\n...\n</think>\n\nanswer`

was parsed with generated whitespace stripped from `reasoning_content` and `content`. The issue is not specific to the v4 branch. The shared reasoning parser on main has the same behavior, and it can be reproduced with the existing `qwen3` reasoning parser.

## Modifications

- Preserve generated whitespace in non-streaming reasoning parsing.
- Preserve generated whitespace in streaming reasoning chunks.
- Avoid stripping content before tool-call boundaries so tool-call payloads keep the model-generated format.
- Add parser unit tests for streaming and non-streaming whitespace preservation.
- Add an OpenAI chat response test to ensure `reasoning_content` and `content` keep the expected newlines.
