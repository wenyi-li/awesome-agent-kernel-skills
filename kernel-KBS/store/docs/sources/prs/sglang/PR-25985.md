## Summary

## Motivation
Fix visual corruption in Wan VAE decoding when Conv3d weights are stored in `channels_last_3d` memory format.

This PR makes Wan causal Conv3d inputs match the memory format of the Conv3d weights before running the convolution, and restores the output back to contiguous format afterward to keep the downstream tensor layout unchanged.

When serving Wan2.2 I2V with 8 GPUs and offload disabled, the generated video could become visually corrupted / garbled on current main.

Reproduction command:

```bash
sglang serve \
    --model-path /home/admin/Wan2.2-I2V-A14B-Diffusers \
    --dit-cpu-offload false \
    --dit-layerwise-offload false \
    --text-encoder-cpu-offload false \
    --image-encoder-cpu-offload false \
    --vae-cpu-offload false \
    --pin-cpu-memory false \
    --num-gpus 8 \
    --port 3000
```
