## Summary

## Motivation

When \`SGLANG_OPT_CACHE_SWA_TRANSLATION=True\`, \`DeepSeekV4TokenToKVPool\` caches the full→SWA index translation in \`self.cached_loc\` to avoid redundant GPU gathers. However, \`register_mapping()\` replaces \`full_to_swa_index_mapping\` without clearing \`cached_loc\`, so subsequent translations can return stale results — for example after a HiCache commit/load-back that rebuilds the SWA mapping.

Note: \`_should_cache_swa\` is \`False\` by default so this only affects users who explicitly set \`SGLANG_OPT_CACHE_SWA_TRANSLATION=True\`.

## Modifications

- \`deepseek_v4_memory_pool.py\`: clear \`self.cached_loc = None\` in \`register_mapping()\` when the mapping tensor is replaced.
