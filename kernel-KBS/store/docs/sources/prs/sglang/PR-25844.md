## Summary

Adds a top-level `kv_events` field to the `/server_info` HTTP response so KV-aware routers can discover each worker's KV-event publisher endpoint via worker introspection instead of operator-side port coordination. The value is a structured descriptor (or `null` when publishing is disabled / misconfigured).

The change is **strictly additive**: no existing `/server_info` field is touched, and the scheduler-side `disaggregation/kv_events.py` is untouched. New logic lives on `ServerArgs` (the config object knows how to describe its own publisher) and reuses the pre-existing `KVEventsConfig.from_cli` parser.
