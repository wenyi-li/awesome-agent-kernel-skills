## Summary

## Motivation

This PR is part of the Q2 LoRA roadmap in https://github.com/sgl-project/sglang/issues/25095, specifically the plan to benchmark and improve multi-LoRA inference.

The current LoRA benchmark helper scripts hard-code `NUM_LORAS = 4`. This makes it awkward to benchmark multi-LoRA serving with different adapter counts. This PR makes the number of benchmark LoRA adapters configurable from the command line while preserving the existing default behavior.

## Modifications

- Add `--base-model-path`, `--lora-path`, and `--num-loras` to `benchmark/lora/launch_server.py`.
- Add `--base-model-path` and `--num-loras` to `benchmark/lora/lora_bench.py`.
- Remove the benchmark client's import dependency on launcher constants.
- Use `--num-loras` to sample benchmark requests from `lora0`, `lora1`, etc.
- Record `base_model_path`, `base_only`, and `num_loras` in benchmark JSON output.
