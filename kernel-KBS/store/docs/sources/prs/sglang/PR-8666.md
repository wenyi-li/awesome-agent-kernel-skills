## Summary

## Motivation


This PR fixes several issues encountered when running the LLaMA 3.2 11B vision model with image tasks on CPU.



## Modifications



- Fixed kv cache access logic for cross attention
  - Affected both backends: 
      - python/sglang/srt/layers/attention/torch_native_backend.py 
      - python/sglang/srt/layers/attention/intel_amx_backend.py
  - Updated the CPU kernel to support such case:
      -  sgl-kernel/csrc/cpu/extend.cpp
      -  sgl-kernel/csrc/cpu/decode.cpp
      -  sgl-kernel/csrc/cpu/torch_extension_cpu.cpp
- Removed hard-coded CUDA device in python/sglang/srt/models/mllama.py
- Fixed incorrect attribute access of mm_inputs in python/sglang/srt/models/mllama.py
- Fixed unrecognized chat template parsing in python/sglang/bench_serving.py
- Supported >2D inputs for weight_packed_linear in python/sglang/srt/layers/quantization/unquant.py
