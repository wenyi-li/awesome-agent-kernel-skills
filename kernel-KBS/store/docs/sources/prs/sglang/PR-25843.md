## Summary

- Route CUDA `concat_mla_k` / `concat_mla_absorb_q` runtime call sites through `sglang.jit_kernel.concat_mla`.
- Keep the MUSA `concat_mla_k` path on the existing `sgl_kernel` AOT implementation.
- Remove the unused JIT `downcast_fp8` wrapper, CUDA header, and JIT-only benchmark.
- Keep `mla_kv_pack_quantize_fp8` and MXFP8 JIT kernels untouched.

## Rationale

`concat_mla` already has a JIT implementation with JIT-vs-AOT coverage. The CUDA runtime paths can use that implementation directly, reducing active dependency on the duplicate AOT CUDA op while leaving the MUSA AOT path intact.

`downcast_fp8` has no SGLang runtime import or call site. A reference scan on `origin/main` only finds the JIT wrapper/header/benchmark themselves; current runtime FP8 KV-cache paths use other Triton/backend-specific implementations instead.
