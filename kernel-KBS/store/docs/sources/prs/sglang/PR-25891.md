## Summary

This moves diffusion pipeline execution into `current_platform.inference_mode()` at the executor boundary for both single-output and grouped execution paths.

The executor now scopes the version-counter fallback at stage granularity:

- Normal resident and layerwise-only stages stay under inference tensor mode.
- Stages that touch FSDP or CPU-offloaded components temporarily run under `torch.inference_mode(False), torch.no_grad()` because those hooks need tensor version counters.
- Layerwise offload prefetch/release creates and swaps normal tensors even when the surrounding request is in inference mode.

## Impact

- No model loading, dtype, or residency policy changes.
- Existing stage profiling still wraps the same execution region.
- FSDP/CPU-offload paths keep the previous `no_grad` safety behavior.
- Pure layerwise DiT/encoder/VAE stages can still benefit from inference tensor mode instead of forcing the whole request back to `no_grad`.
