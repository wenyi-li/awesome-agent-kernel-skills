## Summary

## Motivation

Removes the \`set_swa_loc\` side channel from \`SWAKVPool\` and makes the pool translate \`out_cache_loc\` to SWA pool indices internally, with a memoised per-batch cache.

**Before**: callers pre-set \`pool.swa_loc\` before each \`set_kv_buffer\`, and \`out_cache_loc_swa\` was a pre-computed field on \`ForwardBatch\` that needed to be sliced and threaded through every graph runner.

**After**: \`SWAKVPool.set_kv_buffer\` calls \`translate_loc_from_full_to_swa(loc)\` directly.

### Design

The translation follows a **prepare → fallback** pattern:

1. **Preparation** — \`invalidate_loc_cache()\` is called once per batch before the model forward (in \`_forward_raw()\` and at the top of every \`run_once()\`). This resets the cache so the upcoming forward sees a clean slate.

2. **Translation as fallback** — \`translate_loc_from_full_to_swa()\` fires lazily on the first SWA-layer access within the forward. It computes the gather, stores the result keyed on \`(data_ptr, numel)\`, and every subsequent SWA layer in the same pass gets an O(1) cache hit. The translation is never eagerly pre-computed — it only runs if a SWA layer is actually reached.
