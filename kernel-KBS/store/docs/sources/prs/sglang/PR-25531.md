During LoRA+MoE prefill, `_add_lora_gate_up_delta` calls `.any().item()` on a GPU tensor every layer to check whether any LoRA adapter is active. This forces a `cudaStreamSynchronize` on **every MoE layer, every step**, creating GPU pipeline bubbles that are invisible to kernel-time profiling but clearly visible in Perfetto traces as idle gaps.

On a 48-layer Qwen3-30B-A3B model (TP=4, BS=128, input_len=2048), this produces:
- **260 `cudaStreamSynchronize` calls** per forward pass
- **~100 ms of cumulative GPU stall time** (the first sync alone is ~70 ms as it drains the entire pending GPU stream)
- Visible GPU bubbles between every MoE layer

## Modifications
