## Summary

Workers registered with the disagg cluster storage before their uvicorn HTTP server was ready to accept connections. This caused the disagg server's clock sync (NTP-style GET /steady_clock_offset) to fail, leading to misaligned timestamps and "Cluster is not ready" errors in multi-node service discovery.

Move register_worker() from the FastAPI lifespan startup (which runs before uvicorn binds) into __call__, polling server.started before registering.
