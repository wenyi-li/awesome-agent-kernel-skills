## Description

When the outer K8s pod retry (runKubernetesPodWithInfraRetry) fires for a SLURM stage and spawns a new dispatcher pod, the inner SLURM retry counter inside that new pod resets to 1 and computes postTag="". The Jenkins controller's GlobalState.uploadResultStageNames still carries the dead pod's recorded upload under the same key, so ensureStageResultNotUploaded trips with "Upload test results for X failed because it has already been uploaded." (Observed in builds 2218 / 2153 for several DGX_H100-PyTorch stages; root cause is a TRTLLMINF-43 follow-up gap from PR #13530.)
