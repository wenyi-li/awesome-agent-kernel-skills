## Description

`LLM.sleep()` and `LLM.wakeup()` previously had no dispatch path for the MPI executor. Any caller using `LLM._collective_rpc("sleep", ...)` on an MPI-backed executor would fail silently or raise an `AttributeError`. This PR closes that gap for single-GPU (TP=1, PP=1) deployments.

This PR adds `sleep()`/`wakeup()` to `BaseWorker` so that the MPI executor paths (IPC via `GenerationExecutorWorker`, RPC via `RpcWorker`) can release and restore GPU virtual memory without requiring the Ray executor. Also adds `collective_rpc()` to both MPI proxy classes s
