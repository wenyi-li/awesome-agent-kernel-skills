## Summary

Two wideep disagg perf configs (gb200_wideep_deepseek-{r1,v32}-fp4_1k1k_ctx2_gen1_dep48_bs16_eplb288_mtp3_con12288_ccb-NIXL.yaml) had 'mtp3' in their filenames but were missing the speculative_config block, so the servers were not actually running MTP. Add MTP decoding_type with num_nextn_predict_layers=3 to both gen and ctx workers so the configs match their filenames. PR #14347 will then auto-detect spec decoding from speculative_config.decoding_type and drop --ignore-eos for these cases, keeping MTP acceptance rate measurements accurate.
