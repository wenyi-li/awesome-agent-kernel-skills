## Summary

This PR reduces host-side overhead in the PyTorch TRTLLM/trtllm-gen attention path by moving more attention setup work from Python into fused C++/nanobind ops, caching layer-static backend state, and simplifying the Python call surface. Unsupported configurations continue to fall back to `thop.attention`.

## Main Optimizations

- Caches one `FlashInferTrtllmGenAttention` backend per `TrtllmAttention` layer instead of reconstructing backend state per call.
- Refactors the large `trtllm_gen` callsite to pass `AttentionForwardArgs` plus metadata-derived state, with static layer fields cached on the backend.
- Caches positive `is_supported()` results for stable model/layer configuration.
- Moves workspace layout/materialization from Python into C++ fused ops.
- Adds shared C++ attention workspace layout helpers and unit tests.
- Fuses trtllm-gen context/generation preprocessing through nanobind ops, reducing Python tensor slicing, offset arithmetic, and Python-to-C++ transitions.
- Moves FlashInfer TRTLLM-Gen KV metadata construction into C++ via `build_trtllm_gen_kv_cache_metadata`.
- Keeps FlashInfer shared paged index disabled to match TRT-LLM's current sep
