## Summary
- **Root cause**: Autotuner profiling failures for CUTLASS MoE GEMM tactics on SM103 (GB300) leave pending CUDA errors uncleaned, which propagate as CUBLAS_STATUS_EXECUTION_FAILED to subsequent cuBLAS calls in the forward pass.
- **Fix**: Add torch.cuda.synchronize() wrapped in try/except in the _profile_runners exception handler to explicitly clear any pending CUDA errors after a failed profiling attempt.
