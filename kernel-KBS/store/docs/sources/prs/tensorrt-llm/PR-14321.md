## Summary

indexer_k_dtype is already populated inside indexer_config by update_sparse_attention_indexer_config (which sources it from sparse_attention_config.indexer_k_dtype, falling back to the "fp8" default). Passing it again as an explicit kwarg to DeepSeekV4SparseAttentionConfig while also expanding **indexer_config triggers a duplicate keyword argument TypeError during executor init. Drop the explicit kwarg and let **indexer_config provide the value.
