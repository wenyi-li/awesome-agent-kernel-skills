## Description

This PR continues the DeepSeek V4 scratch reuse work from https://github.com/NVIDIA/TensorRT-LLM/pull/14085 and optimizes KV cache block-table handling for sparse attention paths.

- Makes DSV4 SWA scratch reuse opt-in and refines cache-manager metadata for per-layer/sliding attention block tables.
- Reuses staging buffers for DSV4/KVCacheManagerV2 page table copies and moves `copy_batch_block_offsets` / `copy_batch_sliding_block_tables` transform work onto CUDA tensors after asynchronous host-to-device copies.
- Updates DSV4 cache manager tests to run the GPU copy paths on a CUDA stream and copy results back to CPU for assertions.
