## Summary

Improves CUTLASS Flash Attention (FMHA) attention on Blackwell through two key optimizations: fixing a bug in causal attention trip count calculation that caused wasted computation on empty tiles, and implementing skip correction rescaling in FA4 when the scaling factor equals 1.0. Together these yield up to 1.5x speedup for long-context attention.

## Problem

The CUTLASS FMHA implementation had two attention issues on Blackwell: (1) incorrect `get_unmasked_trip_count` calculation in causal attention masking caused the kernel to process empty tiles, wasting SM cycles; (2) unnecessary correction rescaling operations were performed even when the scaling factor was 1.0, adding overhead in the FA4 mainloop.

## Solution / Techniques

- Fixed causal attention trip count calculation in `flash-attention_fusion.hpp` to correctly skip empty tiles
- Added skip correction rescaling optimization in `sm100_flash-attention_fwd_mainloop_tma_warpspecialized.hpp` when scale == 1.0
- The causal mask fix eliminates wasted cycles on tiles that contain no valid attention entries
- The rescaling skip removes unnecessary floating-point operations in the FA4 warp-specialized mainloop

## Key Code

```cpp
// Skip correction rescaling when scaling factor is 1.0
// in sm100_flash-attention_fwd_mainloop_tma_warpspecialized.hpp
if (scale != 1.0f) {
    // Apply correction rescaling
    apply_correction_rescale(acc, scale);
}
// Saves ~60 TFLOPS/s overhead for long context
```

## Performance

- Skip correction optimization: 60 TFLOPS/s improvement for long context scenarios
- Causal attention fix: approximately 1.5x acceleration for long context by eliminating wasted computation on empty tiles
- Combined effect significantly improves FA4 throughput on Blackwell for causal attention workloads
