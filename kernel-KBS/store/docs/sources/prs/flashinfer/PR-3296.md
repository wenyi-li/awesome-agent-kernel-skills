## Summary

PR #2805 refactored the monolithic CuTe-DSL MLA decode kernel into a modular structure and removed the original implementation. The original authors want it kept available because the modular path is still maturing. This PR restores it under the existing \`backend=\"cute-dsl\"\` user surface (no new backend name) and exposes implementation selection via a new \`cute_dsl_impl=\` keyword argument on \`trtllm_batch_decode_with_kv_cache_mla\`.

- **\"auto\"** (default): monolithic by default, automatically promoted to modular when the call uses a modular-only feature (currently \`sinks\`).
- **\"modular\"**: strict, always run the modular kernels.
- **\"monolithic\"**: strict, always run the monolithic kernels; raises \`ValueError\` if the call uses any modular-only feature.

The dispatcher strips modular-only kwargs (\`sinks=None\`) before forwarding to monolithic, so callers can pass \`sinks=\` unconditionally without breaking the monolithic path.
