## Summary

Adds a GEMM kernel overlapped with kernel-fusion kernel-fusion operations using multimem instructions within the CuTe DSL framework. This fuses kernel-fusion communication with computation to reduce latency in tensor-parallel inference on Blackwell GPUs, leveraging SM100's GDC (Global Data Crossbar) and TMA multicast capabilities.

## Problem

In tensor-parallel LLM inference, GEMM operations followed by kernel-fusion communication create a sequential bottleneck. The GEMM must complete before kernel-fusion can begin, wasting GPU cycles. A fused GEMM+kernel-fusion kernel can overlap computation and communication, reducing end-to-end latency for kernel-fusion inference.

## Solution / Techniques

- Implements GEMM + kernel-fusion kernel-fusion in CuTe DSL (CUTLASS Python DSL)
- Uses multimem instructions for efficient multi-GPU memory access
- Two-shot kernel-fusion: first shot reduces partial results, second shot broadcasts
- Leverages SM100 GDC for cross-SM data movement during kernel-fusion
- TMA multicast for efficient data distribution across SMs
- Single fused kernel eliminates GEMM-kernel-fusion synchronization overhead
