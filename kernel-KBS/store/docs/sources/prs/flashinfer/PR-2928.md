## Summary

FlashInfer PR #2928 clamps `enable_pdl=True` to `False` on devices below SM90 for the CuTe DSL norm path. The affected norm APIs could otherwise request PDL on Ampere, causing generated PTX to contain grid dependency control instructions that require Hopper-or-newer targets.

## Kernel-KBS Lessons

- Gate PDL/GDC by compute capability before compiling or launching CuTe DSL kernels.
- On Ampere (`sm80`, `sm86`, `sm87`), treat explicit `enable_pdl=True` as unsupported unless the call site has a separate Hopper-or-newer dispatch path.
- Preserve auto-detection behavior for callers that pass `None`, but make explicit opt-in safe on unsupported hardware.
- Add architecture guards near the public API boundary so JIT-generated PTX cannot select instructions outside the target ISA.
