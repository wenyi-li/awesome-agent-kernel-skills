## Summary

Introduces TGV GEMM (Tensor-core-accelerated General-purpose Vector GEMM) as a custom BF16 GEMM backend optimized for minimum-latency inference workloads on Blackwell. Achieves up to 60% speedup over cuBLAS on B200 for small problem sizes by leveraging SM100-specific features including tensor memory (tmem) and programmatic dependent launch (PDL).

## Problem

cuBLAS performs suboptimally for small matrix sizes common in LLM decode (small batch, large hidden dimension). The kernel launch overhead and generic optimization strategies of cuBLAS do not exploit Blackwell-specific hardware features like tensor memory and PDL that can dramatically reduce latency for these workloads.

## Solution / Techniques

- Custom CUDA kernels using CUTLASS framework optimized for minimum latency
- Leverages Blackwell-specific tensor memory (tmem) for fast register-like storage
- Programmatic Dependent Launch (PDL) support via `-DCUTLASS_ENABLE_GDC_FOR_SM100=1` flag
- Configurable tile sizes and DMA stages for workload-specific autotuning
- BFloat16 data type optimization targeting decode-phase inference
- Unit tests and benchmarking infrastructure included
