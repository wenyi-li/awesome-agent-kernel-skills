## Summary

FlashInfer PR #844 added a CuTe SM80 MLA decode path. The PR used asymmetric warp roles to handle register-file pressure: more warps were assigned to the output MMA stage while the attention MMA, softmax, and data-load stages kept separate roles.

## Kernel-KBS Lessons

- On SM80 decode kernels, register pressure can be relieved by changing warp-role allocation rather than shrinking every tile immediately.
- Treat output MMA, attention MMA, softmax, and data-load stages as separate scheduling resources.
- Keep specialized SM80 decode kernels behind explicit JIT or dispatch logic because the schedule is architecture- and shape-specific.
