## Summary

## Summary

Adds TinyGEMM as a supported `mm_bf16` backend and includes it in `backend="auto"` candidate selection when BF16 output is requested.

This lets `mm_bf16(..., backend="auto", pdl=True)` consider TinyGEMM instead of excluding it from the BF16 autotuned path. TinyGEMM remains restricted to BF16 output, matching the existing `tinygemm_bf16` API.

## Notes

TinyGEMM is appended after the existing BF16 backends in heuristic order, so it becomes an autotune candidate without becoming the default no-cache fallback for all BF16 auto calls.
