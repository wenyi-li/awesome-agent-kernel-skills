## 📌 Description

This PR wires `trtllm_fmha_v2_prefill` into the `flashinfer_benchmark` harness and fixes several CUDA-graph-incompatible allocations on the FMHAv2 prefill hot path.

### Benchmark integration

- Add `trtllm-fmha-v2` as a backend option for `BatchPrefillWithPagedKVCacheWrapper` and `BatchPrefillWithRaggedKVCacheWrapper` in `benchmarks/routines/attention.py`.
- Register the backend in `flashinfer_benchmark_utils.py` compute-capability map for SM90 and SM120.
- Add it to the refcheck reference priority list (cudnn > cudnn-native > trtllm-gen > trtllm-native > trtllm-fmha-v2).
- For paged runs, thread a separate `kv_token_indptr` (cumulative token counts) since TRT-LLM expects token-based indptr while FlashInfer's paged wrapper uses page-based.
- For paged runs, `kv_cache.contiguous()` is only materialized when `trtllm-fmha-v2` is in the backend list (FlashInfer's `as_strided` HND presentation does not match the physical layout the kernel expects).
