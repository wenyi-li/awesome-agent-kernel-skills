## Summary

Enables the SplitK optimization technique and fixes the tile-scheduling for TensorRT-LLM's fused MoE FP4 implementation in FlashInfer. SplitK partitions the K dimension of GEMM across multiple thread blocks for improved parallelism, while the tile-scheduling fix ensures correct configuration selection for FP4 fused MoE on Blackwell hardware.

## Problem

The TRT-LLM fused MoE FP4 kernel in FlashInfer lacked SplitK support, limiting parallelism for workloads with large K dimensions and small M dimensions common in decode. Additionally, the tile-scheduling had reliability issues when selecting optimal kernel configurations for FP4 fused MoE, leading to suboptimal attention or incorrect configuration choices on SM100.

## Solution / Techniques

- Enables SplitK across all TRT-LLM MoE FP4 variants for improved K-dimension parallelism
- Fixes tile-scheduling robustness for FP4 fused MoE configuration selection
- Enables autotuning in TRT-LLM FP4 tests with enhanced reliability
- Updates header files and synchronizes paths to latest BMM implementation
- Updated metadata hashes to reflect the modifications
- Adds automated testing for the tile-scheduling to prevent regressions

## Performance

SplitK improves throughput for decode-phase MoE computation where M (batch) dimension is small but K (hidden) dimension is large. The tile-scheduling fix ensures optimal kernel configuration is consistently selected for FP4 fused MoE on Blackwell hardware, preventing attention regressions from incorrect configuration choices.
