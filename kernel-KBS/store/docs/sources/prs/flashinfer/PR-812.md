## Summary

FlashInfer PR #812 generalized the MLA template so A100 can vary `CTA_TILE_KV` and `NUM_STAGES`. For Ampere, this is evidence that MLA performance should be tuned through tile-K and pipeline-stage search instead of assuming one fixed template.

## Kernel-KBS Lessons

- Treat A100 MLA as a parameterized template problem: `CTA_TILE_KV` and `NUM_STAGES` are first-class tuning axes.
- Benchmark stage count together with tile size because additional stages consume shared memory and can reduce residency.
- Keep the A100 path explicit instead of folding it into Hopper or Blackwell decode assumptions.
